Clarification

Chatbot vs Agent

From Part 1, some audience confused LLM chatbot with LLM agent. Let's clarify the key distinction.

LLM Chatbot

The LLM chat API responds once per message you send.

It can only answer your question — one turn at a time.

The user drives the conversation

LLM Agent

An agent drives the chat messages forward by itself.

It decides what to do next, calls tools, observes results, and continues.

The agent drives the conversation

The Key Question

Who Drives the Conversation?

The core difference is about control flow.

Chatbot Flow

User sends message → LLM responds → Done

User-driven: The user initiates each interaction

Agent Flow

User sets a goal → Agent loops (think → act → observe) → Goal reached

Agent-driven: The agent autonomously pursues the goal

Key Insight

The Agent Idea

The whole agent concept is about how and who should drive the messages to push toward the goal.

Remember

• Chatbot: User drives, LLM responds once
• Agent: Agent drives, loops until goal is reached
• The "loop" is what makes an agent an agent
